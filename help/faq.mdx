<Accordion title="为什么“自动获取”失败？">
  NextNav 创建、编辑卡片的“自动获取”功能仅仅针对常规普通网站，如果该网站采用的是现代 JS 框架或者设置了反爬防火墙，那么自动获取时极有可能出现失败。

  采用现代 JS 框架的网站需用无头浏览器等方式爬取，将极大的增加 CPU 和内存占用；设置了反爬防火墙的网站需要针对目标写大量针对性爬取代码（有的反爬防火墙针对的是地域，在国外 VPS 部署获取失败，在国内 VPS 部署可能正常）。
</Accordion>

<Accordion title="为什么重新安装后，用已购买的激活码再次激活时失败？">
  重新安装后，用已购买的激活码再次激活必须满足：域名、邮箱、激活码三者与之前保持一致。

  所以，如果域名没有变化，再次激活前请修改邮箱为之前激活时绑定的邮箱。
</Accordion>

<Accordion title="我可以使用同一个邮箱部署多个 NextNav 实例吗？">
  可以。
</Accordion>

<Accordion title="NetNav 实例支持多用户吗?">
  不支持。NextNav 是为了帮助您集中管理浏览器书签，解决跨设备、跨平台、跨浏览器之间同步和访问困难的难题。我并不认为其他用户可以放心的把自己的书签存储到您的 NextNav 实例上。
</Accordion>

<Accordion title="Where is my data stored?">
  All of your bookmarks are stored in a SQLite database file at `data/db.sqlite` on your server. NextNav never sends your data to any external service — everything stays on the machine you control.

  <Tip>
    Back up this single file to protect all of your bookmarks. See the backup question below for details.
  </Tip>
</Accordion>

<Accordion title="How do I access NextNav on my phone?">
  Open your NextNav URL in any mobile browser. The interface is fully responsive and adapts to smaller screens automatically. For faster access, add NextNav to your home screen as a Progressive Web App (PWA):

  <Steps>
    <Step title="Open NextNav in your mobile browser">
      Navigate to your NextNav URL in Safari (iOS) or Chrome (Android).
    </Step>
    <Step title="Open the browser share or menu">
      Tap the Share button on iOS or the three-dot menu on Android.
    </Step>
    <Step title="Add to Home Screen">
      Select **Add to Home Screen** and confirm. NextNav will appear as an app icon on your home screen.
    </Step>
  </Steps>
</Accordion>

<Accordion title="Can I use NextNav as a browser new-tab page?">
  Yes. Most desktop browsers let you set a custom URL as your new-tab page. Set it to your NextNav URL and your bookmarks will appear every time you open a new tab.

  <Tabs>
    <Tab title="Chrome">
      Install a new-tab extension such as **Custom New Tab URL**, then enter your NextNav URL in the extension settings.
    </Tab>
    <Tab title="Firefox">
      Go to **about:config**, search for `browser.newtab.url`, and set the value to your NextNav URL.
    </Tab>
    <Tab title="Edge">
      Go to **Settings** → **New tab page** → **Custom** and enter your NextNav URL.
    </Tab>
  </Tabs>
</Accordion>

<Accordion title="Does NextNav work offline?">
  NextNav requires a network connection to the server it's running on. If you host it locally (for example, on your home server or laptop), it works fine on your home network without any internet connection. You only need internet access if your server is hosted remotely.
</Accordion>

<Accordion title="What browsers are supported?">
  NextNav works in any modern browser. Confirmed supported browsers include:

  - **Chrome** (and Chromium-based browsers)
  - **Firefox**
  - **Microsoft Edge**
  - **Safari** (desktop and iOS)
  - **Brave**

  <Note>
    Very old browser versions may have limited support. Keep your browser up to date for the best experience.
  </Note>
</Accordion>

<Accordion title="How do I back up my bookmarks?">
  You have two options for backing up your bookmarks:

  **Option 1 — Copy the database file (recommended)**

  Copy `data/db.sqlite` from your server to a safe location. If you're using Docker, the file lives in the mounted volume at `/data/nextnav/db.sqlite` by default.

  ```bash
  cp /data/nextnav/db.sqlite ~/nextnav-backup-$(date +%F).sqlite
  ```

  **Option 2 — Export as HTML**

  Log in as admin, open the **Admin Panel**, and use the export feature to download your bookmarks as a standard browser-compatible HTML file. You can re-import this file into NextNav or any browser.
</Accordion>

<Accordion title="Can I run NextNav on a Raspberry Pi or NAS?">
  Yes. NextNav is intentionally lightweight — the Docker image is under 100 MB — and it runs on ARM-based devices including Raspberry Pi (all models with 64-bit OS) and most NAS devices that support Docker.

  <Tip>
    Use the `linux/arm64` or `linux/arm/v7` Docker image tag to match your device's architecture.
  </Tip>
</Accordion>

<Accordion title="How do I update to a new version?">
  Pull the latest Docker image and recreate your container. Your data is safe as long as you're using a persistent volume.

  ```bash
  docker pull imawolff8/nextnav:latest
  docker stop nextnav && docker rm nextnav
  # Re-run your original docker run command
  ```

  For the full update walkthrough, including Docker Compose instructions, see the [Docker deployment guide](/deployment/docker).
</Accordion>
