This page covers the most common issues people run into with NextNav and walks you through exactly how to resolve each one. Work through the steps in order — most problems have a simple root cause.

<Accordion title="I can't reach NextNav in my browser">
  If the page times out or refuses to connect, work through these checks:

  **1. Confirm the container is running**

  ```bash
  docker ps | grep nextnav
  ```

  If you don't see a `nextnav` entry, the container has stopped. Check the logs to find out why:

  ```bash
  docker logs nextnav --tail 50
  ```

  **2. Verify the port mapping**

  Make sure the host port you chose (for example, `3080`) is not blocked by your server's firewall. On Linux, run:

  ```bash
  sudo ufw status
  # or
  sudo firewall-cmd --list-ports
  ```

  Open the port if it's blocked, then try again.

  **3. Confirm the address you're using**

  - Use the server's local IP address (e.g., `http://192.168.1.50:3080`), not `localhost`, when accessing from another device.
  - If you've set up a domain name, verify your DNS record or `/etc/hosts` entry is correct.
</Accordion>

<Accordion title="The page loads but shows a blank white screen">
  A blank page almost always means a PHP error is silently preventing rendering.

  **Check the container logs first:**

  ```bash
  docker logs nextnav
  ```

  Look for any `PHP Fatal error`, `Warning`, or `Permission denied` messages.

  **Verify the `data/` directory is writable:**

  NextNav needs write access to its `data/` directory to read and write the SQLite database. Fix permissions with:

  ```bash
  chmod -R 755 /data/nextnav
  ```

  **For manual (non-Docker) installs, check your PHP version:**

  ```bash
  php -v
  ```

  NextNav requires PHP 7.4 or higher. If you're on an older version, upgrade PHP before running NextNav.
</Accordion>

<Accordion title="I forgot my admin password">
  <Warning>
    Before attempting any database reset, back up your `data/db.sqlite` file. Resetting the database permanently deletes all of your bookmarks.
  </Warning>

  Try these options in order:

  **Option 1 — Follow the password reset guide**

  See [User Management](/configuration/user-management) for the standard password reset steps using the admin panel or CLI.

  **Option 2 — Reset via the database (Docker)**

  Exec into the running container and update the password hash directly in SQLite:

  ```bash
  docker exec -it nextnav sh
  sqlite3 /var/www/html/data/db.sqlite
  ```

  Then run the appropriate `UPDATE` statement to replace the password hash. Refer to the [User Management](/configuration/user-management) page for the exact query.

  **Option 3 — Last resort: trigger first-run setup**

  Rename or delete `data/db.sqlite` to force NextNav to run its initial setup wizard on next load. You will lose all bookmarks — only do this if you have a backup or nothing to recover.

  ```bash
  mv /data/nextnav/db.sqlite /data/nextnav/db.sqlite.bak
  ```
</Accordion>

<Accordion title="Bookmark import fails or shows no bookmarks after import">
  Work through these checks to diagnose a failed import:

  **Confirm the file format is HTML**

  NextNav's import feature expects a standard browser bookmark export in HTML format. JSON exports (used by some password managers and third-party tools) are not supported.

  **Use a supported browser export**

  Export your bookmarks directly from Chrome, Firefox, or Edge using each browser's built-in bookmark manager export option. Third-party export tools may produce non-standard HTML that NextNav cannot parse.

  **Test with a smaller file**

  Export a single folder instead of your entire bookmark library and try importing that first. If it succeeds, your full export file may be malformed — try re-exporting it.

  **Check the logs for PHP errors during import**

  ```bash
  docker logs nextnav --tail 100
  ```

  Look for any errors that appear at the timestamp when you attempted the import.
</Accordion>

<Accordion title="The browser extension can't connect to NextNav">
  If the extension reports a connection error or fails to save bookmarks, check each of these:

  **Verify NextNav is reachable from your browser**

  Open your NextNav URL directly in the same browser. If that fails, fix the connectivity issue first (see the first troubleshooting entry above).

  **Check your SSL certificate**

  If your NextNav instance uses HTTPS, the SSL certificate must be valid and trusted by your browser. A self-signed certificate will cause the extension to reject the connection. Use a certificate from Let's Encrypt or your CA of choice.

  **Remove any trailing slash from the URL**

  In the extension settings, make sure the NextNav URL ends without a trailing slash:

  - ✅ `https://nav.example.com`
  - ❌ `https://nav.example.com/`

  **Check your network**

  If you normally access NextNav via a local IP address, that address won't work when you're away from home (e.g., when your VPN is not active). Use your public domain name or enable your VPN before using the extension remotely.
</Accordion>

<Accordion title="Drag-and-drop sorting isn't working">
  Drag-and-drop bookmark sorting has a few requirements — verify all of them:

  **Switch to the `default2` theme**

  Drag-and-drop is only available in the `default2` theme. To switch:

  <Steps>
    <Step title="Open the Admin Panel">
      Log in as admin and click **Admin Panel** in the navigation.
    </Step>
    <Step title="Go to Settings">
      Select **Settings** from the admin menu.
    </Step>
    <Step title="Change the theme">
      Set the **Theme** dropdown to `default2` and save.
    </Step>
  </Steps>

  **Make sure you're logged in as admin**

  Guest users cannot reorder bookmarks. Confirm you're logged in with the admin account before attempting to drag.

  **Click the icon, not the text**

  To initiate a drag, click and hold the bookmark's **icon** (the favicon on the left side of the bookmark). Clicking the bookmark text or title will follow the link instead.
</Accordion>

<Accordion title="The NextNav container keeps restarting">
  A container that restarts in a loop almost always has a startup error. Diagnose it with:

  ```bash
  docker logs nextnav --tail 50
  ```

  **Most common cause: missing or incorrect data volume**

  If the host directory mapped to the container's `/var/www/html/data` path doesn't exist, the container will fail to start. Create it and restart:

  ```bash
  mkdir -p /data/nextnav
  docker restart nextnav
  ```

  If you're using Docker Compose, verify the `volumes:` path in your `docker-compose.yml` matches an existing directory on your host. After correcting it, run:

  ```bash
  docker compose down && docker compose up -d
  ```
</Accordion>

<Accordion title="Changes in the admin panel don't appear on the front page">
  If you save a change but the front page still shows the old content, the issue is almost always caching.

  **Hard-refresh your browser**

  Force your browser to reload all assets without using the cache:

  | OS | Shortcut |
  |----|----------|
  | Windows / Linux | `Ctrl + Shift + R` |
  | macOS | `Cmd + Shift + R` |

  **Flush your reverse proxy cache**

  If NextNav sits behind a reverse proxy (Nginx, Caddy, Traefik, etc.) with caching enabled, flush the proxy's cache after making changes. For example, with Nginx proxy cache:

  ```bash
  rm -rf /var/cache/nginx/*
  nginx -s reload
  ```

  <Note>
    For day-to-day use, consider disabling caching on your reverse proxy for the NextNav upstream to avoid stale content.
  </Note>
</Accordion>
