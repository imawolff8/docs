If you already run Apache or Nginx with PHP on your server, you can install NextNav by dropping the application files into your web root — no Docker or containerization required. This approach fits shared hosting environments and existing LAMP or LEMP stacks where you want to keep everything under one web server.

## Requirements

Make sure your environment meets all of the following before you begin:

- **PHP 7.4 or higher** — NextNav uses syntax and features not available in older PHP versions.
- **SQLite 3 extension enabled** — NextNav stores all data in a local SQLite database. Confirm the extension is active by checking `php -m | grep sqlite3`.
- **Apache or Nginx web server** — any recent stable version works.
- **Write permissions on the web root** — the application needs to create and update files in the `data/` directory at runtime.

## Installation

<Steps>
  <Step title="Download the latest release">
    Go to the NextNav releases page and download the latest archive:

    ```
    https://github.com/imawolff8/NextNav/releases
    ```

    Download the `.zip` or `.tar.gz` asset from the most recent release tag.
  </Step>
  <Step title="Extract the archive to your web server root">
    Unpack the downloaded file into your desired installation directory. For example:

    ```bash
    unzip nextnav-*.zip -d /var/www/html/nextnav
    ```

    Adjust the destination path to match your server's document root or virtual host configuration.
  </Step>
  <Step title="Set write permissions on the data directory">
    The `data/` directory must be writable by the web server process:

    ```bash
    chmod -R 755 /var/www/html/nextnav/data
    ```

    If your web server runs as a different user (for example `www-data` on Debian/Ubuntu systems), also set the owner:

    ```bash
    chown -R www-data:www-data /var/www/html/nextnav/data
    ```
  </Step>
  <Step title="Run the setup wizard">
    Open your browser and navigate to your NextNav URL — for example:

    ```
    http://your-server/nextnav/
    ```

    NextNav detects a fresh installation and launches the setup wizard automatically. It initializes the SQLite database and prompts you to create an admin account. Complete the wizard before using the application.
  </Step>
</Steps>

## Web Server Configuration

### Nginx

Add a `location` block to your server configuration to route all requests through NextNav's front controller:

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/html/nextnav;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_pass unix:/run/php/php8.1-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }

    location ~ /\.ht {
        deny all;
    }
}
```

Reload Nginx after saving the file: `sudo systemctl reload nginx`.

### Apache

NextNav ships with an `.htaccess` file in the application root. Enable `mod_rewrite` and ensure `AllowOverride All` is set for your document root directory in your Apache virtual host or main config — the `.htaccess` file handles the rest automatically.

```apacheconf
<Directory /var/www/html/nextnav>
    AllowOverride All
    Require all granted
</Directory>
```

Reload Apache after making changes: `sudo systemctl reload apache2`.

<Note>
  NextNav's SQLite database file is stored inside the `data/` directory. Make sure you include this folder in your regular server backups. Losing the `data/` directory means losing all your saved bookmarks.
</Note>

<Tip>
  For most users, Docker deployment is simpler to maintain and update. If you'd like to switch, see the [Docker deployment guide](/deployment/docker) — your bookmarks can be migrated using the built-in export tool.
</Tip>
