By the end of this guide you'll have a fully running NextNav instance, a configured admin account, and your first bookmark saved — all in five minutes or less. The steps below use Docker, which is the fastest path to a working installation on any machine.

<Note>
  You need one of the following before you begin:

  - **Docker** (recommended) — Docker Desktop on macOS or Windows, or Docker Engine on Linux. Any recent version works.
  - **PHP 7.4 or higher** with SQLite enabled — if you prefer a manual install on an existing web server, follow the [Manual Install](/deployment/manual) guide instead.
</Note>

## Steps

<Steps>
  <Step title="Pull and run the NextNav Docker image">
    Run the following command in your terminal. It downloads the image and starts the container in the background:

    ```bash
    docker run -itd \
      --name="nextnav" \
      -p 3080:80 \
      -v /data/nextnav:/data/wwwroot/default/data \
      --restart always \
      helloz/onenav
    ```

    Docker maps port `3080` on your host to the container, and mounts `/data/nextnav` so your bookmarks persist across restarts and updates. The container starts automatically on reboot thanks to `--restart always`.
  </Step>
  <Step title="Open the setup wizard">
    Open your browser and navigate to:

    ```
    http://localhost:3080
    ```

    NextNav's first-run setup wizard loads automatically. Follow the on-screen prompts to choose an admin username and password. Complete the wizard before proceeding.
  </Step>
  <Step title="Log into the admin panel">
    After finishing the wizard, go to the login page:

    ```
    http://localhost:3080/index.php?c=login
    ```

    Enter the admin credentials you just created. You'll land on the main dashboard.
  </Step>
  <Step title="Save your first bookmark">
    Click **Add Bookmark** in the bottom toolbar. Fill in the URL and an optional title, then click **Save**. Your first link now appears in the dashboard — accessible from any device on your network.
  </Step>
</Steps>

<Tip>
  Ready to move beyond a local test setup? See [Deploy with Docker](/deployment/docker) for production-grade configuration including reverse proxy and HTTPS.

  Already have a library of browser bookmarks? Head to [Import & Export](/usage/import-export) to bring them all in at once instead of adding them one by one.
</Tip>
