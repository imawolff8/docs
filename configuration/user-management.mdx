NextNav uses a single administrator account — the account you create during first-run setup. There are no roles, teams, or additional user seats to manage. Every administrative action, from adding bookmarks to changing settings, goes through this one account.

## Logging In

To access your NextNav admin account, navigate to the login page directly:

```
http://YOUR_SITE/index.php?c=login
```

Replace `YOUR_SITE` with your NextNav domain or IP address. Enter the username and password you set during the initial setup wizard, then click **Log In**.

<Note>
  If you deployed NextNav behind a reverse proxy or with a custom base path, adjust the URL accordingly. The login path `index.php?c=login` is always relative to your NextNav root.
</Note>

## Changing Your Password

<Steps>
  <Step title="Log in to your NextNav instance">
    Navigate to `http://YOUR_SITE/index.php?c=login` and sign in with your current credentials.
  </Step>
  <Step title="Open the Admin Panel">
    Click **Admin Panel** in the bottom toolbar of the dashboard.
  </Step>
  <Step title="Go to Account Settings">
    Select **Account Settings** from the Admin Panel navigation menu.
  </Step>
  <Step title="Enter your current password">
    Type your existing password in the **Current Password** field to authorize the change.
  </Step>
  <Step title="Set your new password">
    Enter and confirm your new password in the provided fields. Choose a strong, unique password.
  </Step>
  <Step title="Save the change">
    Click **Save**. Your new password is active immediately on the next login.
  </Step>
</Steps>

## Resetting a Forgotten Password

If you've lost access to your admin account, you need to reset the password directly through the database. The method depends on how you deployed NextNav.

<Warning>
  These steps require shell access to your server or container. Proceed carefully — incorrect database edits can corrupt your NextNav data. Always back up `data/db.sqlite` before making any manual changes.
</Warning>

<Tabs>
  <Tab title="Docker">
    <Steps>
      <Step title="Open a shell inside the container">
        Run the following command on your host machine, replacing `nextnav` with your actual container name if different:

        ```bash
        docker exec -it nextnav /bin/sh
        ```
      </Step>
      <Step title="Locate the database">
        The SQLite database is stored at `/var/www/html/data/db.sqlite` inside the container (or the path mapped by your volume mount).
      </Step>
      <Step title="Update the password hash">
        Open the database with the SQLite CLI and update the `password` field in the `users` table with a new bcrypt hash:

        ```sh
        sqlite3 /var/www/html/data/db.sqlite
        ```

        ```sql
        UPDATE users SET password = 'YOUR_BCRYPT_HASH' WHERE username = 'admin';
        ```

        Generate a bcrypt hash using a tool like [bcrypt-generator.com](https://bcrypt-generator.com) or a local utility. Use a cost factor of 10 or higher.
      </Step>
      <Step title="Exit and log in">
        Type `.quit` to exit SQLite, then `exit` to leave the shell. Log in through the browser using your new password.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Manual Install">
    <Steps>
      <Step title="Locate the database file">
        Navigate to your NextNav installation directory. The database is at:

        ```
        data/db.sqlite
        ```
      </Step>
      <Step title="Back up the database">
        Copy the file before making any changes:

        ```bash
        cp data/db.sqlite data/db.sqlite.bak
        ```
      </Step>
      <Step title="Update the password hash">
        Open the database with the SQLite CLI and update the admin password with a new bcrypt hash:

        ```bash
        sqlite3 data/db.sqlite
        ```

        ```sql
        UPDATE users SET password = 'YOUR_BCRYPT_HASH' WHERE username = 'admin';
        .quit
        ```
      </Step>
      <Step title="Log in with the new password">
        Return to the login page and sign in using your updated credentials.
      </Step>
    </Steps>

    **Alternative — delete the database to trigger first-run setup:**

    If you don't need to preserve your data, you can delete `data/db.sqlite` entirely. NextNav will treat the missing file as a fresh install and present the setup wizard on next visit.

    <Warning>
      Deleting the database to reset your password removes ALL your bookmarks and settings permanently. Back up `data/db.sqlite` first if you want any chance of recovery.
    </Warning>
  </Tab>
</Tabs>

## Guest Access Control

NextNav lets you set individual bookmarks as **private** (visible to admins only) or **public** (visible to unauthenticated visitors). This gives you fine-grained control over what guests can see.

- **Public bookmarks** — visible to anyone who visits your NextNav URL, even without logging in.
- **Private bookmarks** — only visible when you are logged in as admin.

You can also disable guest access entirely at the site level. Go to **Admin Panel → Settings → Guest Access** and toggle the setting off. When guest access is disabled, unauthenticated users are redirected to the login page and cannot view any bookmarks, regardless of their individual visibility setting.

<Tip>
  For a private personal bookmark manager, disable guest access entirely in Settings so unauthenticated users see nothing. This is the simplest way to keep your bookmarks to yourself without managing per-bookmark visibility.
</Tip>
